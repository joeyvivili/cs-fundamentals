# CS61A Practice Code Repository

## Overview

This folder contains my personal practice and exploration code for CS61A. It serves as a working space to deepen understanding of core concepts such as data abstraction, recursion, higher-order functions, interpreters, and SQL.

The goal is not just to complete assignments, but to **experiment, compare different approaches, and build intuition** about how things work under the hood.

---

## Contents

### 📁 `scheme_editor/`

A workspace for writing and testing Scheme code.
Used for:

* Interpreter-related experiments
* Macro exploration
* Practicing recursive patterns

---

### 🐍 `ex.py`

Python practice file.

Includes:

* Implementations of data structures (e.g., `Link`, `Tree`)
* Multiple approaches to the same function
* Experiments with recursion, generators, and higher-order functions
* Debugging and edge case testing

---

### 🧠 `ex.scm`

Scheme practice file.

Includes:

* Recursive list processing
* Macro definitions (`define-macro`, quasiquote, unquote)
* Interpreter-related transformations (e.g., `let-to-lambda`)
* Experiments with evaluation rules

---

### 🗄️ `ex.sql`

SQL practice file.

Includes:

* Query writing (SELECT, JOIN, aggregation)
* Practice with CS61A-style database problems
* Testing edge cases and query variations

---

### ⚙️ SQL Interpreter File

Used to run and test SQL queries.

Typical workflow:

* Write queries in `ex.sql`
* Execute them through the interpreter
* Debug schema and query logic

---

## Purpose & Approach

This folder is designed for:

* **Exploration over memorization**
* **Understanding over completion**
* **Comparing multiple solutions**

Common activities include:

* Rewriting the same function in different styles (recursive vs iterative)
* Testing edge cases
* Breaking and fixing code to understand failure modes
* Translating ideas across languages (Python ↔ Scheme ↔ SQL)
